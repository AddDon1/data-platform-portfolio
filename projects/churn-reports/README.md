# ОТТОК - Скрипт для анализа эффективности маркетинговых кампаний

[![Python](https://img.shields.io/badge/Python-3.7+-blue.svg)](https://www.python.org/)

Этот проект демонстрирует архитектуру и логику обработки данных для анализа эффективности маркетинговых кампаний. Скрипт обрабатывает данные о клиентах, их транзакциях и бонусах, формируя целевые и контрольные группы для последующего анализа.

> **Важно**: Этот файл создан специально для демонстрации в портфолио. В нем сохранена вся логика обработки данных, но с заменой реальных учетных данных и API endpoints на демонстрационные значения.

## 📖 Описание

Скрипт ОТТОК (Отчет о доходном оттоке) предназначен для:

- Анализа поведения клиентов после участия в маркетинговых кампаниях
- Формирования целевых групп клиентов на основе API данных
- Создания контрольных групп для A/B-тестирования
- Расчета метрик эффективности (сумма транзакций, бонусы, количество чеков)

Проект демонстрирует навыки работы с:
- REST API интеграцией
- Обработки больших объемов данных с использованием pandas
- Логирования и обработки ошибок
- Структурированного кода с clear separation of concerns

## ✨ Возможности

- **Автоматизация получения данных**: Скрипт автоматически получает список клиентов из целевых групп через API
- **Формирование контрольных групп**: Алгоритм отбирает клиентов с похожими характеристиками, исключая участников кампаний
- **Мультипериодный анализ**: Обработка данных за несколько месяцев с расчетом метрик за разные периоды
- **Гибкая настройка**: Легко адаптируется под разные типы кампаний и метрики
- **Логирование и мониторинг**: Подробное логирование всех этапов обработки
- **Генерация отчетов**: Сохранение результатов в CSV формате для дальнейшего анализа

## 🏗️ Архитектура

Скрипт построен по принципу модульности с четким разделением ответственности:

```
ottok.py
│
├── Основные функции
│   ├── clear_target_control_folders()     # Очистка директорий
│   ├── get_token()                         # Аутентификация
│   ├── get_clients_list()                  # Получение клиентов из API
│   ├── get_target_group_clients()          # Формирование целевых групп
│   ├── get_dates_of_accrued_bonuses()      # Анализ бонусов
│   ├── get_clickhouse_data()               # Транзакционные данные
│   ├── get_control_group_data()            # Формирование контрольной группы
│   ├── process_campaign()                  # Обработка кампании
│   ├── concat_csv_files()                  # Объединение результатов
│   └── main()                              # Главная точка входа
│
└── Логика обработки
    ├── Target Group (Целевая группа)
    │   └── Клиенты, которые участвуют в кампании
    │
    └── Control Group (Контрольная группа)
        └── Клиенты с похожими характеристиками, не участвующие в кампании
```

## 🔄 Как это работает

### 1. Получение токена аутентификации

```python
token = get_token()
```
Скрипт получает токен доступа через OAuth2 с парой логин/пароль.

### 2. Получение клиентов из целевых групп

```python
get_target_group_clients(token)
```
- Запрашивает список клиентов из API для Push-кампании (ID: 834)
- Запрашивает список клиентов из API для Mail-кампании (ID: 833)
- Формирует глобальные множества для последующей обработки

### 3. Анализ бонусов

```python
clients_with_dates = get_dates_of_accrued_bonuses(clients, date_end)
```
Определяет, какие клиенты получили бонусы в указанный период.

### 4. Получение транзакционных данных

```python
data_chl = get_clickhouse_data(clients_with_dates, date_end)
```
Извлекает метрики по транзакциям:
- Сумма транзакций за 7 и 30 дней после даты начисления
- Начисленные и потраченные бонусы
- Количество чеков

### 5. Формирование контрольной группы

```python
control_group = get_control_group_data(clients_with_dates, date_end, limit, folder)
```
Алгоритм отбирает клиентов для контрольной группы:
- Исключает участников целевых групп
- Исключает клиентов из других контрольных групп
- Исключает клиентов из "Высокодоходных" кампаний
- Фильтрует по пороговым значениям (минимум 30 000 за 90 дней, 0 за 30 дней)
- Ограничивает размер контрольной группой (1/8 от целевой)

### 6. Генерация отчетов

```python
concat_csv_files(folder)
```
Объединяет все сгенерированные CSV-файлы в единые отчеты:
- `push_target_group.csv` - все клиенты целевой группы push
- `push_control_group.csv` - все клиенты контрольной группы push
- `mail_target_group.csv` - все клиенты целевой группы mail
- `mail_control_group.csv` - все клиенты контрольной группы mail

from ottok import process_campaign, get_token, get_target_group_clients

process_campaign(target_group_id=834, folder='push', date_end_list=['20260301', '20260302'])

```python
from ottok import get_clickhouse_data, get_dates_of_accrued_bonuses

# Получение клиентов с бонусами на определенную дату
clients_with_dates = get_dates_of_accrued_bonuses(

# Получение транзакционных данных
data = get_clickhouse_data(clients_with_dates, '20260301')

```

## 📊 Формат выходных данных

### Структура CSV-файлов

**Целевая группа (target_group):**
| person_uid | date_acc | sum_30_days | sum_90_days | sum_next_7_days | bp_next_7_days | ba_next_7_days | ch_cnt_next_7_days | sum_next_30_days | bp_next_30_days | ba_next_30_days | ch_cnt_next_30_days |
|------------|----------|-------------|-------------|-----------------|----------------|----------------|---------------------|------------------|-----------------|-----------------|---------------------|
| UUID123    | 20260301 | 15000       | 45000       | 5000            | 500            | 300            | 15                  | 12000            | 1200            | 800             | 35                  |

**Контрольная группа (control_group):**
| person_uid | close_date | sum_30_days | sum_90_days | sum_next_7_days | bp_next_7_days | ba_next_7_days | ch_cnt_next_7_days | sum_next_30_days | bp_next_30_days | ba_next_30_days | ch_cnt_next_30_days |
|------------|------------|-------------|-------------|-----------------|----------------|----------------|---------------------|------------------|-----------------|-----------------|---------------------|
| UUID456    | 20260301   | 0           | 50000       | 4500            | 400            | 250            | 12                  | 10000            | 1000            | 700             | 30                  |

**Пояснение колонок:**
- `person_uid` - Уникальный идентификатор клиента
- `date_acc` / `close_date` - Дата транзакции
- `sum_30_days` / `sum_90_days` - Сумма транзакций за предыдущие периоды
- `sum_next_7_days` / `sum_next_30_days` - Сумма транзакций после кампании
- `bp_next_*` - Бонусы потраченные
- `ba_next_*` - Бонусы начисленные
- `ch_cnt_next_*` - Количество чеков

## 🔒 Безопасность

Демонстрационный скрипт не содержит реальных учетных данных:

- ✅ Заменены реальные API endpoints на `api.example.com`
- ✅ Удалены реальные пароли и токены
- ✅ Заменены имена серверов баз данных
- ✅ Удалены специфичные идентификаторы кампаний

**Важно**: Для production использования необходимо:
1. Создать файл `config.py` с реальными параметрами подключения
2. Добавить `config.py` в `.gitignore`
3. Использовать переменные окружения для хранения секретов
