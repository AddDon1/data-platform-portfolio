# Action Chance - Логика расчёта купонов акции "Сладкий холод"

## Описание

`action_chance.py` — это демонстрационная версия скрипта для автоматизации процесса начисления купонов участникам акции "Сладкий холод". Скрипт реализует полный цикл обработки данных: от загрузки информации о участниках и их покупках до генерации купонов и их отправки на сайт акции через API.

Версия для портфолио содержит обобщённые данные и имитацию взаимодействия с внешними системами для демонстрации архитектуры и логики на собеседованиях.

## Основные возможности

- ✅ Получение списка участников акции
- ✅ Выгрузка данных о покупках из баз данных (ClickHouse/OLAP-куб)
- ✅ Расчёт количества купонов по условиям акции
- ✅ Генерация уникальных номеров купонов
- ✅ Проверка на дубликаты
- ✅ Генерация отчётов в Excel и JSON
- ✅ Создание отчётов для пуш-уведомлений
- ✅ Отправка купонов на сайт через API

## Условия акции

- **Название**: Сладкий холод
- **Период**: 2026-06-11 00:00:00 — 2026-07-08 23:59:59
- **Минимальная сумма чека**: 800 ₽
- **Критерий для франшиз**: минимум 3 покупки в неделю

## Архитектура

Скрипт следует модульному принципу с разделением на логические шаги:

```
1. setup_directories()        — Создание структуры папок
2. load_registered_cards()    — Загрузка зарегистрированных карт
3. get_participant_info()     — Получение информации о участниках
4. get_purchase_data()        — Выгрузка данных о покупках
5. calculate_coupons()        — Расчёт количества купонов
6. generate_coupons()         — Генерация уникальных номеров
7. check_duplicates()         — Проверка на дубликаты
8. create_push_reports()      — Создание отчётов для пушей
9. post_coupons_api()         — Отправка купонов на API
10. save_reports()            — Сохранение финальных отчётов
```

## Как использовать

### Запуск демонстрационной версии

```bash
python action_chance.py
```

Скрипт выполнит весь цикл обработки с симуляционными данными и создаст отчёты в папке `reports/`.

### Основные функции

#### `setup_directories()`
Создаёт необходимые папки (`input_files/`, `reports/`, `logs/`), если они не существуют.

#### `load_registered_cards()`
Загружает список зарегистрированных карт из внешнего источника (в демо-версии — создаёт симуляционные данные). В реальном скрипте данные берутся с FTP-сервера.

#### `get_participant_info()`
Получает информацию об участниках акции (номер карты, телефон, ФИО) из OLAP-куба Loymax через DAX-запросы.

#### `get_purchase_data(participants_df)`
Выгружает данные о покупках из ClickHouse по следующим источникам:
- Основные магазины (фрешмаркеты)
- Франшизы (Командор №13, 14, 213, 91)
- Онлайн-заказы

Фильтрует чеки по:
- Датам акции
- Минимальной сумме (800 ₽)
- Количеству покупок в неделю для франшиз

#### `calculate_coupons(df_purchases, participants_df)`
Рассчитывает количество купонов для каждого участника:
- Для фрешмаркетов: `floor(sum(summ)/800)` или 1 (в зависимости от условий)
- Для франшиз: фиксированная логика (всегда 1 купон за чек при соблюдении условий)
- Для онлайн: отдельная логика начисления

#### `generate_coupons(df_with_coupons)`
Генерирует уникальные номера купонов, начиная с 10000 (формат: 00001, 00002, ...). Проверяет на дубликаты с существующими купонами в базе.

#### `check_duplicates(df_coupons)`
Проверяет уникальность сгенерированных номеров купонов и сравнивает их с уже существующими в системе.

#### `create_push_reports(df_purchases, df_participants)`
Создаёт два отчёта для пуш-уведомлений:
- `push_1.xlsx` — участники, которые зарегистрировались, но не выполнили условия
- `push_2.xlsx` — участники, которые выполнили условия, но не зарегистрировались

#### `post_coupons_api(df_coupons)`
Отправляет купоны на API сайта акции (`https://api.example.ru/example_api/v1/coupons/`):
1. Получает токен авторизации
2. Формирует JSON-payload
3. Отправляет POST-запрос
4. Обрабатывает ошибки с повторами

#### `save_reports(df_with_coupons, df_coupons)`
Сохраняет финальные отчёты:
- Excel-файл с участниками и количеством купонов
- JSON-файл для импорта на сайт

## Технологии

- **Pyadomd** — взаимодействие с OLAP-кубом Loymax (в реальном скрипте)
- **requests** — HTTP-запросы к API
- **BeautifulSoup4** — парсинг HTML-ответов

## Примечания

- Демонстрационная версия использует жёстко заданные симуляционные данные
- В реальном скрипте все данные загружаются из внешних систем в реальном времени
- Логика расчёта купонов может отличаться в зависимости от условий конкретной акции
