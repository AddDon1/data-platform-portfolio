# Персональные предложения месяца (ППМ)  
# ML-пайплайн для рекомендаций 2+ млн клиентов  
# 📊 AUC = 0.82, CTR ↑18%, охват = 99.2%  
# 🛠️ CatBoost, ClickHouse, MariaDB, API интеграции, чанкинг (100k)

НАЧАЛО ПАЙПЛАЙНА
    0. ПОДГОТОВКА И КОНФИГУРАЦИЯ
        - Настройка логирования
        - Определение конфигурации:
            • дата-маркер (например, 202605),
            • пути к данным и бэкапам,
            • параметры модели,
            • параметры fallback-стратегии

    I. ОБУЧЕНИЕ И ВАЛИДАЦИЯ МОДЕЛИ (раз в квартал или при изменении данных)
        1. ФОРМИРОВАНИЕ ОБУЧАЮЩЕГО ДАТАСЕТА
            - Для каждого месяца (в течение года):
                • Выборка клиентов: с фильтрацией по сегментам (например, по квантилям трат)
                • Формирование признаков:
                    - временные: месяц покупок, частота, сезонность и тд.
                    - поведенческие: средние траты, средний ценовой сегмент товаров, частота, формат и тд.
                    - товарные: средняя цена, средний срок годности, еда/не еда и тд.
                    - демографические: пол, возраст, маркеры (пенсионер, мобильное приложение) и тд.
                • Формирование таргета:
                    - целевая категория = та:
                        a) которую клиент покупал реже всего по кол-ву месяцев покупок за отчетный период
                        b) ИЛИ показала «высокую сезонность» (>1.5)
            - Конкатенация датасетов по месяцам → единый датасет для обучения

        2. ОБУЧЕНИЕ МОДЕЛИ
            - Использование CatBoostClassifier:
                • scale_pos_weight для баланса классов (для редкого target=1)
                • GPU-ускорение, ранняя остановка
                • cat_features = none (все категориальные уже one-hot)
            - Валидация:
                • Accuracy, AUC-ROC, precision-recall, F1-score на hold-out
                • Confusion matrix для анализа ошибок (TP / FP / TN / FN)

        3. СОХРАНЕНИЕ МОДЕЛИ
            - Сохранение модели в формате `.cbm`
            - Сохранение scaler-а (StandardScaler) для инференса

    II. ОСНОВНОЙ ПАЙПЛАЙН (для клиентов с достаточной историей)
        1. СБОР ДАННЫХ О КЛИЕНТАХ
            - Получение списка активных клиентов (person_uid) за последние 3 месяца + 1 месяц год назад
            - Разбиение на чанки по 100k клиентов (для масштабируемости)

        2. ФОРМИРОВАНИЕ ПРИЗНАКОВ ДЛЯ ПРЕДСКАЗАНИЯ
            - Для каждого чанка:
                • Формирование аналогичных признаков, что и при обучении
                • Генерация файлов (по одному на чанк)

        3. МАШИННОЕ ОБУЧЕНИЕ (инференс)
            - Загрузка сохранённой модели и scaler-а
            - Предсказание:
                • predicted_class (0 / 1)
                • predicted_proba (вероятность класса 1)
            - Сохранение результатов

        4. ЗАГРУЗКА В БАЗУ ДАННЫХ (предсказания)
            - Очистка временной таблицы 'predict'
            - Вставка только рекомендуемых (predicted_class = 1)
            - Столбцы: uid, group_name, group_id, base_sum, predicted_proba

        5. ГЕНЕРАЦИЯ ИТОГОВОГО СПИСКА (рекомендации)
            - Запрос данных из 'predict' → финальный список рекомендаций 
            - Объединение:
                * ML-рекомендации,
                * Брендовые рекомендации (таблица 'brand_predict'),
                * Универсальные рекомендации (fallback для новых или редких клиентов)
            - Разбиение по 100k уникальных uid

        6. ИНТЕГРАЦИЯ (API)
            - Формирование JSON с учётом модального канала и ППМ:
                • 9 категорий с атрибутами: название, картинка, период действия,
                  preferenceValue (процент кэшбека), description
            - Разбиение JSON на части (по 6000 записей) → папка «готовых к импорту»
            - Импорт во внешнюю систему через API

    III. FALLBACK-СТРАТЕГИЯ (для новых/редких клиентов)
        - Проверка: отсутствуют ли uid в 'predict'
        - Применение правил: ABC + сезонность → 9 категорий
        - Интеграция в общий поток

    IV. АУДИТ И ЛОГИРОВАНИЕ
        - Сколько рекомендаций из ML / fallback / ошибок
        - Проверка: JSON не пустые, размер ≤6000, поля заполнены
